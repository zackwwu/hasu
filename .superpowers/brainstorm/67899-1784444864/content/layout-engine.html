Layout Engine Algorithm

How tiles get placed — symmetry, balance, and no sliver cuts

The Core Problem

Surface: 2000mm × 1200mm | Tile: 300mm × 200mm | Grout: 3mm
❌ Naive: start from left edge
  Full tiles: 2000 ÷ 303 = 6 tiles (1818mm)
  Remainder: 182mm cut on right → 182/300 = 61% tile ← okay
  But: asymmetric. No cut on left, big cut on right.
✅ Good: center the layout
  Shift start by: 182 ÷ 2 = 91mm
  Left cut: 91mm (30% tile)  |  Right cut: 91mm (30% tile)
  Symmetric + no slivers.
⚠️ Sliver detected
  If remainder were 60mm → 60/300 = 20% tile → TOO THIN
  Algorithm falls back: remove one full tile, redistribute.
  2000 - (5 × 303) = 485mm → 242mm each side → 81% tile each → good.

The Algorithm (per direction, per tile group region)


function computeLayout(regionW, regionH, tileW, tileH, groutW, groutH):
  unitW = tileW + groutW
  unitH = tileH + groutH

  // Step 1: Fit tiles
  fullCols = floor(regionW / unitW)
  fullRows = floor(regionH / unitH)
  remainderW = regionW - (fullCols * unitW)
  remainderH = regionH - (fullRows * unitH)

  // Step 2: Check for slivers (cut < 30% of tile)
  minCutRatio = 0.30
  if remainderW < tileW * minCutRatio AND remainderW > 0:
    fullCols -= 1    // drop one tile, redistribute
    remainderW = regionW - (fullCols * unitW)
  // same for height

  // Step 3: Center for symmetry
  offsetX = remainderW / 2
  offsetY = remainderH / 2

  // Step 4: Place tiles in grid
  tiles = []
  for row in 0..fullRows:
    for col in 0..fullCols:
      x = offsetX + col * unitW
      y = offsetY + row * unitH
      w = min(tileW, regionW - x)  // clamp at edge
      h = min(tileH, regionH - y)
      isCut = (w < tileW) or (h < tileH)
      tiles.add({x, y, w, h, isCut})

  return {tiles, offsetX, offsetY}

Multi-Group Surfaces

Each SurfaceTileGroup defines a rectangular region on the surface. The layout engine runs independently per region:

  Surface (2000 × 1200)
  ├── Region A: {x:0, y:0, w:2000, h:150} → border tile, brick pattern
  ├── Region B: {x:0, y:150, w:2000, h:900} → subway tile, grid pattern
  └── Region C: {x:0, y:1050, w:2000, h:150} → border tile, brick pattern

Regions don't overlap. Layout engine treats each independently, then the Render Engine stitches them together on the Canvas. Grout alignment across regions (stretch goal) would require a second pass that nudges region offsets to align grout lines.
