Interaction Model

Step 1: Select a surface

  • Tap a surface in the sidebar or tap it in the 3D room view
  • Selected surface gets a blue border
  • Its 2D elevation fills the main canvas
  • Blue drag handles appear on edges

Step 2: Drag the layout

  • One-finger drag anywhere on canvas
  • Entire tile grid shifts with finger
  • Wrap-around: tiles exiting left reappear on right
  • Offset {x, y} updates live in the info overlay
  • Cut tiles recalculate in real-time

Step 3: Lock + drag together

  • Toggle lock on other surfaces in sidebar
  • Drag selected surface → locked parallel surfaces shift by same delta
  • Perpendicular surfaces: lock has no effect (axes don't match)
  • Use case: keep opposite walls symmetric
  • Lock state is transient, not saved to project

What Drag Actually Changes

Dragging modifies SurfaceTileGroup.offset — the starting position of the tile grid relative to the surface origin.

  Original:  offset = {x: 91, y: 50}   ← computed by layout engine (centered)
  After drag: offset = {x: 103, y: 45}  ← user shifted right 12mm, up 5mm

This triggers LayoutResult.stale = true, then a debounced recompute (~100ms after finger lifts). The layout engine reruns with the new offset, recalculates cut tiles, and the canvas repaints.

Fine-tuning doesn't change tile sizes or pattern. It only shifts the grid origin. The tiler is moving the same tiles to a better position, not redesigning the layout.
