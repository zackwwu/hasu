Fine-Tuning: Lock Propagation Rules

How deltas apply to locked surfaces — depends on the axis

The Rule

Drag on selected wall Propagates to parallel locked walls Propagates to perpendicular locked walls
Horizontal (dx)
shift tiles left/right along width 		✅ same dx
same axis direction 		❌ no effect
horizontal axes point in different 3D directions
Vertical (dy)
shift tiles up/down along height 		✅ same dy ✅ same dy
all walls share the same "up" in 3D space

Why This Matters in Practice

Use case: align horizontal grout lines across all walls

  • Lock all walls together
  • Drag one wall vertically → all walls shift vertically by the same dy
  • Grout lines stay at the same height around the room
  • This is a partial solve of the stretch goal

Use case: keep opposite walls symmetric

  • Lock front and back walls (parallel)
  • Drag one horizontally → both shift horizontally together
  • Vertical drag also propagates (shared up)
  • Result: symmetrical layout on both sides

Floors

Floors have no shared axis with walls (floor surface is horizontal, wall surface is vertical). Locking between a wall and a floor has no effect for MVP. Each surface type (wall, floor) only locks within its own type.

Drag on floor Propagates to locked floor
Horizontal (dx — width)✅ if same orientation
Vertical (dy — depth)✅ if same orientation
