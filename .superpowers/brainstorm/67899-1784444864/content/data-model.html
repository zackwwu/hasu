Data Model

SQLite tables + Dart models

Core Entities

Room → Surfaces → SurfaceTileGroups → TileGroup → Tiles
Room ├── id, name ├── units: "mm" | "in" └── surfaces: List<Surface> Surface (wall or floor) ├── id, type: "wall" | "floor" ├── width, height (mm/in) ├── position: {x, y, z} + rotation (relative to room origin) ├── groutColor: "black" | "grey" | "white" ├── groutWidth: float (default 3mm) └── tileAssignments: List<SurfaceTileGroup> SurfaceTileGroup (join table — many-to-many with layout params) ├── surfaceId, tileGroupId ├── layoutArea: {x, y, width, height} — which region of the surface ├── pattern: "grid" | "herringbone" | "brick" | "stacked" ├── offset: {x, y} — manual fine-tune offset └── locked: bool — for synchronized movement TileGroup ├── id, name (e.g. "White subway", "Blue mosaic border") ├── tileWidth, tileHeight ├── texturePath — captured/imported image file ├── source: "captured" | "imported" └── colorHint — for quick preview without texture loaded LayoutResult (computed, cached) ├── id, surfaceId ├── tiles: List<PlacedTile> │ ├── {x, y, width, height} on surface │ ├── isCut: bool │ ├── cutEdge: "left"|"right"|"top"|"bottom"|null │ └── tileGroupId ├── computedAt: timestamp └── stale: bool — recompute if params changed

Key Design Decisions

SurfaceTileGroup is the join

Each surface can have 1+ tile groups. Each group assigned to a rectangular region of the surface. This enables: border tiles around the edge, feature strips, split walls.

LayoutResult is cached

Tile placement is computed once, stored, only recomputed when params change (tile size, surface dims, offset). This keeps the interactive editor snappy — drag to fine-tune re-renders from cached positions.

Position is relative to room

Each surface has a 3D position + rotation relative to room origin. This enables: rendering all surfaces together in 3D, computing adjacent edges for grout alignment (stretch goal).
